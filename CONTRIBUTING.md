# Вклад в проект

Ниже краткие правила для командной работы.

## Установка
1. Клонировать репозиторий.
```bash
git clone <repo_url>
```
2. Создать виртуальное окружение.

```bash
python -m venv .venv
```
3. Установить проект в editable-режиме:

```bash
pip install -e .
```

## Зависимости
- Все зависимости храним в `pyproject.toml`.
- Добавление зависимостей делаем вручную: правим список `dependencies`.
- Dev-зависимости (линтеры и т.п.) добавляем в `project.optional-dependencies.dev`.

## Ветки и PR
- Каждый контрибутор работает в своей ветке.
- В `main` вливаем только через PR.
- Перед PR прогоняем `ruff`.

## Pre-commit
Если хотите авто-проверки перед коммитом:

```bash
pip install -e .[dev]
pre-commit install
```

После этого `ruff` и форматтер будут запускаться автоматически при `git commit`.

## Импорты
- Внутри модулей пакета используем относительные импорты: `from .utils import ...`.
- В скриптах используем импорты через имя библиотеки: `from pacman_rldp.env import ...`.
- Это нужно, чтобы проект работал как устанавливаемая библиотека.

## Сторонний код
Общее правило для стороннего кода - его складываем только в third_party и используем оттуда. В начале файла в комментарии лучше процитировать, откуда взят код и авторов.

## Логи и результаты
- Экспериментальные результаты не коммитим, они остаются в `results/`.
- Для материалов, которые пойдут в презентацию, используем `results/important/`. Эта папка коммитится.
